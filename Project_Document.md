This project consists of two coordinated Java applications—a Spring Boot–based server and a standalone command-line client—that together form a complete tech-support ticketing platform. The server exposes a rich REST API for creating, assigning, tracking, scheduling, and reporting on support tickets, while the CLI client consumes that API to deliver punch-list reports directly in the terminal. Both applications live in separate GitHub repositories, share the same domain model, and communicate over HTTP so you can develop, test, and version them independently yet deploy and operate them as a unified service.
The server application defines JPA entities for clients, technicians, service types, tickets, appointments, feedback entries, technician skills, and ticket history, all mapped to a relational schema in H2 or MySQL. It implements business rules in a service layer that calculates SLA-based due dates, finds the least-loaded qualified technician for each new ticket, prevents appointment double-bookings, and logs every ticket change into a history table. Controllers expose endpoints for ticket CRUD operations (including bulk close and assign), appointment scheduling, feedback submission, history retrieval, and SLA or utilization reporting. A global exception handler translates validation and business-rule violations into appropriate HTTP statuses. Unit tests, written with JUnit 5 and Mockito, mock repositories to cover every branch of the service logic, and WebMvcTests verify correct controller behavior and JSON mappings. Continuous integration via GitHub Actions runs the full test suite on each push.
